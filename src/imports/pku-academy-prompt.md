# PKU Academy — Вайбкодинг-промпт v2

## Контекст
React/TSX scrollytelling сайт для детей с ФКУ (фенилкетонурия), возраст 6 лет.
Тематика: **спорт** (не космос/топливо). Две игровые миссии.
Стек: React + TypeScript + Tailwind + Framer Motion (предположительно).

## 🔴 ПРИОРИТЕТ 1 — Критичные баги

### 1.1 "Talk Like a Captain" — всё должно влезать в экран

**Проблема:** игра обрезается по высоте, кнопку "Продолжить" невозможно нажать без скролла.

**Решение:**
- Обернуть весь контейнер игры в `height: 100vh` + `overflow: hidden`
- Внутри использовать `display: flex; flex-direction: column; justify-content: space-between`
- Варианты ответов (options list) — добавить `overflow-y: auto; max-height: <вычислить оставшееся место>`
- Кнопка "Продолжить" — всегда `position` в нижней части flex-контейнера, никогда не уходит за экран
- Если вариантов много — уменьшить `gap`, `padding`, `font-size` внутри игры (только внутри этого компонента!)
- Проверить на viewport 768px, 1024px, 1280px — во всех кнопка должна быть видна

```tsx
// Примерная структура компонента:
<div className="h-screen overflow-hidden flex flex-col">
  <div className="flex-shrink-0 p-4">
    {/* Заголовок вопроса */}
  </div>
  <div className="flex-1 overflow-y-auto px-4">
    {/* Варианты ответов */}
  </div>
  <div className="flex-shrink-0 p-4">
    {/* Кнопка Продолжить — ВСЕГДА внизу */}
  </div>
</div>
```

### 1.2 Убрать scroll hijacking, стрелки, snap

**Проблема:** страница останавливается при скролле, есть кнопки-стрелки/skip, страница не скроллится свободно.

**Решение:**
- Найти и удалить: `scroll-snap-type`, `scroll-snap-align`, `overscroll-behavior`, любой JS/событие `wheel` или `touchmove` с `preventDefault()`
- Удалить компоненты кнопок-стрелок навигации (Skip, →, ↓ между секциями)
- Убрать любой `pointer-events: none` на body/html во время анимации
- Страница должна скроллиться нативно и непрерывно, как обычный лендинг
- Framer Motion `whileInView` анимации — **оставить**, они не мешают скроллу

## 🟢 ПРИОРИТЕТ 3 — Структура: разделение миссий

**Проблема:** преподаватель говорит, что ребёнку непонятно что происходит и нет чёткого разделения между миссиями.

**Решение: Mission Splash Screen + Progress Bar**

### 3.1 Splash Screen перед каждой миссией

Добавить полноэкранный компонент `<MissionSplash />` перед секциями каждой миссии:

```tsx
// Компонент MissionSplash
interface MissionSplashProps {
  number: 1 | 2;
  title: string;       // "Собери рюкзак!"
  subtitle: string;    // "Подготовься к тренировке"
  emoji: string;       // "🎒" | "🏆"
  accentColor: string; // CSS-переменная или Tailwind класс
}
```

**Миссия 1:**
- Число: `1`
- Заголовок: `"Миссия 1: Собери рюкзак!"`
- Подзаголовок: `"Собери всё нужное перед тренировкой"`
- Акцент: синий / голубой

**Миссия 2:**
- Число: `2`
- Заголовок: `"Миссия 2: Время матча!"`
- Подзаголовок: `"Покажи что ты знаешь — и вперёд на поле!"`
- Акцент: оранжевый / зелёный

**Анимация Splash Screen** (Framer Motion):
```tsx
// Появляется при whileInView, крупный текст, scale + fade + slide
initial={{ opacity: 0, scale: 0.8 }}
whileInView={{ opacity: 1, scale: 1 }}
transition={{ duration: 0.6, ease: "easeOut" }}
```

Splash Screen занимает `min-h-screen`, центрированный контент.

### 3.2 Sticky Progress Bar

Добавить sticky header (фиксированный сверху) — тонкая полоска с прогрессом:

```
[🏃 Миссия 1: Собери рюкзак] ●●●○○  [Миссия 2: Время матча!]
```

- Высота: ~48px, полупрозрачный фон с blur (`backdrop-blur`)
- Обновляется через `IntersectionObserver` или Framer Motion `useScroll` при входе в секции каждой миссии
- На мобильном: только иконка миссии + номер, без длинного текста
